# Design

Use for design-focused frontend tasks. If implementation intent is present (implement, fix, refactor, patch, edit, run tests), hand off to coding.

For specific design tasks, load the matching skill:

- UI critique / review / feedback / audit -> `skills/design-critique/SKILL.md`
- UI/layout/styling and design direction -> `skills/design-guidelines/SKILL.md`
- Animation / storyboard / motion -> `skills/storyboard-animation/SKILL.md`
- DialKit / sliders / controls / tuning -> `skills/dialkit/SKILL.md`

## Key rules

- Keep responses design-first: clarity, hierarchy, interaction quality, user context.
- Prefer actionable outputs: prioritized issues, concrete edits, copy-paste-ready snippets.
- If a task mixes critique and implementation, start with critique, then provide implementation guidance.
- If the request is ambiguous across modules, ask one brief clarifying question.
